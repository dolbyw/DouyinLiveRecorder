# 阶段一：请求层与工程基线设计

## 目标

在不改动现有平台解析接口和用户运行方式的前提下，完成路线图阶段一：复用 HTTP 连接、默认启用 TLS 校验、建立可诊断的请求错误模型、锁定依赖，并建立最小可执行的质量门禁。

## 范围

本轮修改限定为：

- `src/http_clients/async_http.py`
- `src/http_clients/sync_http.py`
- `src/http_clients/client_pool.py`
- `src/http_clients/errors.py`
- `src/http_clients/__init__.py`
- `main.py` 中请求客户端的启动与关闭生命周期
- `pyproject.toml`、`uv.lock`
- `tests/http_clients/` 与必要的基础测试配置

不在本轮迁移平台适配器、不重构 `spider.py` 的业务逻辑、不改变配置文件格式，也不升级整体并发模型。

## 兼容策略

继续保留 `async_req()`、`async_fetch_response()`、`get_response_status()` 和 `sync_req()` 的主要参数及返回方式，避免一次性修改 `spider.py` 中的大量调用点。

请求执行内核使用显式异常；兼容入口可以在原有调用确实依赖字符串返回时，将异常转换为带根因的文本。新增代码和测试直接使用请求执行内核，从而逐步摆脱“错误字符串等同响应正文”的历史约定。

`abroad` 参数保留，并映射到独立的客户端用途。代理地址仍沿用现有标准化逻辑。

## 客户端池

异步请求池按以下配置键复用 `httpx.AsyncClient`：

- 用途：`direct`、`proxy`、`abroad`
- 代理地址
- TLS 校验开关
- HTTP/2 开关

同步请求使用长生命周期 `requests.Session`，按用途、代理地址和环境代理策略复用。池内部用锁保护创建与移除，避免并发重复构建。

客户端池提供幂等关闭函数：

- 异步关闭函数负责 `await client.aclose()`；
- 同步关闭函数负责 `session.close()`；
- 主程序在正常退出时关闭两类资源；
- 同步会话保留 `atexit` 兜底，但异步客户端不依赖 `atexit` 启动临时事件循环。

关闭后再次发起请求时，池可以创建新客户端，便于测试和受控重启。

## TLS 策略

所有公共请求入口默认 `verify=True`。只有已确认存在历史证书兼容问题的调用点可以显式传入 `verify=False`，且该降级必须局部可见，不能成为全局默认值。

移除同步请求层中默认关闭主机名和证书校验的 SSL 上下文。标准直连和代理请求均使用底层库的默认 CA 验证行为。

本阶段不建立可远程修改的 TLS 白名单，避免扩大不安全配置面；兼容例外直接留在具体平台调用点，并通过代码搜索和测试保持可审计。

## 错误模型

定义统一的 `HttpClientError` 基类，并至少区分：

- 连接失败；
- 请求超时；
- 代理错误；
- HTTP 状态错误；
- 响应解码错误；
- JSON 解析错误；
- 无效 URL 或请求配置错误。

异常保存请求方法、URL、阶段、原始异常和可读详情。请求执行层使用异常链保留根因。HTTP 响应是否调用 `raise_for_status()` 由入口契约明确控制，避免无意改变依赖 4xx 响应正文的平台行为。

日志与兼容字符串必须包含失败类别和原始根因，不输出 Cookie、Authorization 等敏感请求头。

## 请求行为

超时仍允许每次请求覆盖，默认值保持与现有接口兼容。重定向、请求体、Cookie 返回和文本编码行为保持不变。

本阶段不加入自动重试。平台请求通常带有时效性或签名，盲目重试可能掩盖真实故障或重复提交。后续如需重试，应按幂等方法和明确错误类型单独设计。

## 依赖与质量门禁

使用 `uv lock` 生成并提交 `uv.lock`。开发依赖维持 `pytest` 与 `ruff`，必要时只增加测试异步代码所需的最小依赖。

阶段一的本地质量门禁为：

1. `uv run ruff check`；
2. `uv run ruff format --check`；
3. `uv run pytest -q`。

若全仓库已有大量历史 lint 问题，先把 Ruff 检查范围限定到本轮修改文件和测试目录，并记录全仓清理为后续工作，不能用批量格式化制造无关变更。

## 测试设计

测试不访问真实直播平台或公网。使用 `httpx.MockTransport`、请求库 mock 和可注入的客户端构造器验证：

- 相同配置复用同一客户端，不同用途或代理隔离；
- 并发获取不会重复创建客户端；
- 默认启用 TLS 校验，显式例外可以关闭；
- `direct`、`proxy`、`abroad` 正确选择池键；
- 超时、连接、代理、状态及解码错误映射到正确异常类型并保留根因；
- 兼容入口维持所需返回形态；
- 同步和异步关闭均幂等，关闭后可以重新创建；
- 敏感请求头不会出现在错误文本中。

## 验收标准

- 请求热路径不再为每次请求创建新客户端或会话；
- 公共请求入口默认 TLS 校验开启，降级调用点显式且可搜索；
- 请求执行层能分类报告错误并保留原始根因；
- 主程序具备明确、可重复调用的资源关闭流程；
- `uv.lock` 存在且可用于复现安装；
- 阶段一测试全部通过，Ruff 对约定范围检查通过；
- `spider.py` 的现有调用无需大规模迁移，用户配置和启动方式不变。

## 风险控制

默认启用 TLS 可能暴露部分历史平台的证书问题。处理方式是根据测试或真实故障为单个平台增加局部例外，而不是恢复全局不校验。

异步客户端不能跨多个同时运行的事件循环安全共享。当前主程序存在“线程内多次 `asyncio.run()`”模式，因此客户端池必须按事件循环隔离，或在检测到循环变化时创建对应实例；不得把同一个异步客户端跨循环复用。阶段五统一事件循环后再简化这层兼容逻辑。

同步请求当前混用 `urllib` 与 `requests`。本阶段优先统一到可复用的 `requests.Session`，但必须用测试固定 400 响应正文、重定向和编码等历史行为，避免平台回归。
